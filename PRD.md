# 감정 일기 서비스 - 프론트엔드 PRD

## 🧭 개요

- **프로젝트 명**: 감정 일기 서비스
- **목표**: 따뜻한 감정 캐릭터와 함께 감정을 기록하고 시각화하는 개인 맞춤형 감정 일기 서비스
- **디자인 컨셉**: 사용자의 감정에 공감하는 감정 캐릭터가 있는 따뜻하고 친근한 인터페이스
- **범위**: 프론트엔드 구현 (백엔드 제외)

---

## ✅ 주요 기능

### 1. 로그인 기능

#### 1.1 SNS 로그인

- **지원 플랫폼**: Google, Naver, Kakao
- **UI 요구사항**
  - 각 플랫폼 아이콘 및 버튼 스타일 제공
  - 감정 캐릭터가 환영하는 따뜻한 온보딩 화면
- **동작**
  - OAuth 인증 완료 후 토큰 수신
  - 로그인 성공 시 메인 페이지로 리디렉션

#### 1.2 로그인 이력 저장

- **역할**: 로그인 성공 시 사용자 ID 및 시간 정보를 백엔드에 전달

---

### 2. 일기 기능

#### 2.1 나의 일기 이력 보기 (캘린더)

- **형태**: 월간 캘린더 UI
- **표시 규칙**
  - 작성 완료: 해당 감정의 감정 캐릭터 아이콘 표시
  - 미작성: 날짜 숫자만 표시
- **디자인 특징**
  - 감정 캐릭터의 감정 표현이 각 날짜에 작은 아이콘으로 표시
  - 부드러운 색상과 둥근 모서리로 친근한 느낌
- **보안**
  - 본인만 조회 가능
  - 일기 내용은 화면에 표시되지 않음 (암호화 전제)

#### 2.2 일기 작성하기

- **조건**: 당일 날짜만 작성 가능
- **UI**: AI와 질의응답 기반 일기 작성 인터페이스
- **디자인 특징**
  - 감정 캐릭터가 질문하는 형태의 대화형 인터페이스
  - 사용자 응답에 따라 캐릭터가 반응하는 애니메이션
- **요소**
  - AI 질문 (감정 캐릭터가 말하는 형태)
  - 사용자 응답
  - 저장 버튼

#### 2.3 감정 분석

- **타이밍**: 일기 저장 완료 시 자동 실행
- **출력**

  - 일/주 단위 감정 그래프 (Line or Bar Chart)

- **디자인 특징**
  - 감정 캐릭터가 감정 분석 결과를 설명하는 형태

#### 2.4 감정 타임라인 리포트

- **기능**
  - 감정 캐릭터가 제공하는 따뜻한 요약 문구

---

### 3. 감정 캐릭터 시스템

#### 3.1 감정별 캐릭터 표현

- **감정별 캐릭터 모습**: 사용자의 감정에 따라 캐릭터의 표정과 자세 변화 ( 이미지 교체로 표현 )
  - **happy**: 밝고 활발한 표정
  - **sad**: 위로해주는 따뜻한 표정
  - **angry**: 달래주려는 차분한 표정
  - **anxious**: 함께 걱정해주는 공감하는 표정

#### 3.2 감정 캐릭터와의 상호작용

- **기본 상호작용**: 클릭/터치로 감정 캐릭터와 간단한 상호작용
- **위로 시스템**: 부정적 감정 시 캐릭터의 위로 메시지 및 애니메이션
- **격려 시스템**: 긍정적 감정 시 캐릭터의 함께 기뻐하는 반응

---

### 4. 알림 기능

#### 4.1 일기 쓰기 알현

- **알림 조건**
  - 설정한 특정 시간에 알림 전송
  - 일정 기간 미작성 시 작성 유도 푸시 알림
- **UI**
  - 감정 캐릭터가 그리워하는 느낌의 알림 메시지
  - 알림 시간 설정 UI 제공
  - 브라우저 알림 권한 요청 처리

---

### 5. 메인 화면

#### 5.1 최초 로그인 배경

- **요구사항**: 감정 캐릭터와 함께하는 감성적 배경 이미지
- **기술**
  - CSS background-image 또는 `<picture>` 태그 활용
  - 부드러운 페이드 인/아웃 애니메이션

#### 5.2 로그인 후 메인화면

- **레이아웃**
  - 중앙: 메인 감정 캐릭터 (현재 감정 상태 반영)
  - 상단: 간단한 네비게이션 메뉴
  - 하단: 주요 기능 버튼들 (일기 쓰기, 캘린더 보기 등)
- **요구사항**
  - 감정 캐릭터가 전하는 따뜻한 감성 문구 랜덤 출력
  - 사용자의 최근 감정에 맞는 배경 색상/테마 자동 설정

## 🎨 디자인 가이드라인

### 감정 캐릭터 디자인

- **스타일**: 따뜻하고 친근한 일러스트 스타일
- **감정 표현**: 4가지 기본 감정(happy, sad, angry, anxious)에 대한 명확한 표정 차이
- **애니메이션**: 부드럽고 자연스러운 움직임
- **크기**: 메인 화면에서는 크게, 캘린더 등에서는 작은 아이콘 형태

### 전체 UI 디자인

- **색상**: 부드러운 파스텔 톤, 감정별 테마 색상
- **타이포그래피**: 둥글고 친근한 폰트
- **레이아웃**: 여백을 충분히 활용한 깔끔한 디자인
- **모서리**: 둥근 모서리로 부드러운 느낌

### 디자인 키워드

- **따뜻함**: 감정 캐릭터와의 감정적 유대감
- **친근함**: 접근하기 쉬운 인터페이스
- **부드러움**: 둥근 모서리와 파스텔 색상
- **감성적**: 감정에 공감하는 디자인
- **직관적**: 쉽게 이해할 수 있는 인터페이스
- **차분함**: 마음을 편안하게 하는 색감

## 🚀 MVP 기능 우선순위

1. **감정 캐릭터 표시**: 4가지 감정별 감정 캐릭터 이미지 표시
2. **일기 작성**: 기존 AI 질문 시스템 + 캐릭터 UI 적용
3. **캘린더**: 감정별 캐릭터 아이콘이 표시되는 캘린더
4. **메인 화면**: 감정 캐릭터 중심의 대시보드
5. **기본 상호작용**: 감정 캐릭터 클릭 시 간단한 반응

## 💡 핵심 차별화 포인트

1. **감정 친화적 UI**: 펫 캐릭터를 통한 감정 표현의 시각화
2. **따뜻한 사용자 경험**: 차가운 데이터가 아닌 따뜻한 동반자 느낌
3. **직관적 감정 인식**: 펫의 표정만 봐도 감정 상태를 알 수 있음
4. **심리적 안정감**: 혼자가 아닌 펫과 함께한다는 느낌
